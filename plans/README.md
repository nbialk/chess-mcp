- **"CI has no typecheck gate"** — initially reported, then disproven:
  `skybridge build` runs `tsc -b --force` before Vite
  (`node_modules/skybridge/dist/commands/build.js:21-29`), so `pnpm build`
  in CI already gates type errors. Only the lint gap survived (plan 006).
- **PostHog personal API key (`phx_…`) in `opencode.json` / `.mcp.json`** —
  both files are gitignored (`.gitignore:16-17`) and have zero git history;
  local-only MCP auth config, by design. Optional hardening: opencode
  supports `{env:VAR}` interpolation. No rotation needed.
- **SSRF via chess.com archive URLs** (`src/server.ts:250`) — URLs come from
  chess.com's own API response, not user input. By design.
- **`posthog-node` `flushAt: 1, flushInterval: 0`** — intentional for
  scale-to-zero delivery on Cloud Run; shutdown flush exists.
- **SIGTERM handler racing in-flight requests** (`src/server.ts:425-429`) —
  theoretical request drops on deploys; flush is ~ms and Cloud Run grants a
  grace period. Not worth fixing at current traffic.
- **Dependency audit** — `pnpm audit --prod` clean; no major-version lag on
  critical paths.
- **Formatter (Prettier/Biome format)** — deliberately excluded from plan
